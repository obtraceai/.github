# Contributing

Thanks for contributing to Obtrace.

## Before opening a pull request

- check the documentation first
- open an issue for larger feature work
- keep changes scoped to a single repository or concern when possible
- avoid committing secrets, tokens, or private operational details

## Pull request expectations

- describe what changed and why
- include validation steps
- update docs when behavior or public APIs change
- include screenshots for UI changes when relevant

## Repository guide

- public SDK usage belongs in the SDK repositories
- frontend product work belongs in `obtrace/`
- core backend work belongs in `ingest/`
- docs site work belongs in `obtrace-docs-service/`

## Security

Do not disclose vulnerabilities publicly. Use the process in [SECURITY.md](./SECURITY.md).
